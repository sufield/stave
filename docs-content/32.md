## Compliance-Tagged Buckets Must Have Object Lock Enabled

**Control:** CTL.S3.LOCK.001 (medium)

**Misconfiguration:** The bucket is tagged for compliance but Object Lock is not enabled, so data can still be deleted or overwritten. Object Lock provides WORM (Write Once Read Many) protection required by regulatory frameworks.

**Raw AWS CLI commands**

```bash
aws s3api get-object-lock-configuration --bucket audit-records
aws s3api get-bucket-tagging --bucket audit-records
```

**Bad AWS CLI output**

Object Lock not enabled on a compliance-tagged bucket. Save as `lock.json` and `tagging.json`:

```bash
cat > lock.json <<'EOF'
{
  "ObjectLockConfiguration": {}
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "compliance",
      "Value": "hipaa"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind`, `properties.storage.tags.compliance` (present), and `properties.storage.object_lock.enabled eq false`.

```bash
mkdir -p observations

jq -n \
  --argjson lock "$(cat lock.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "audit-records",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "audit-records",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        object_lock: {
          enabled: (($lock.ObjectLockConfiguration.ObjectLockEnabled // "") == "Enabled")
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Object Lock enabled with COMPLIANCE mode retention. Save as `lock_fixed.json`:

```bash
cat > lock_fixed.json <<'EOF'
{
  "ObjectLockConfiguration": {
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Years": 7
      }
    }
  }
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

jq -n \
  --argjson lock "$(cat lock_fixed.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "audit-records",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "audit-records",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        object_lock: {
          enabled: (($lock.ObjectLockConfiguration.ObjectLockEnabled // "") == "Enabled")
        }
      }
    }

}' > observations-after/2026-03-22T000000Z.json

stave apply \
  --observations observations-after \
  --max-unsafe 12h \
  --now 2026-03-22T12:00:00Z
```

**AWS doc basis for the fix**

Object Lock can be enabled on existing buckets (versioning required). Set a default retention:

```bash
aws s3api put-object-lock-configuration \
  --bucket audit-records \
  --object-lock-configuration '{
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Years": 7
      }
    }
  }'
```
