## PHI Buckets Must Use COMPLIANCE Mode Object Lock

**Control:** CTL.S3.LOCK.002 (medium)

**Misconfiguration:** The bucket is tagged `data-classification: phi` and has Object Lock enabled, but the default retention mode is `GOVERNANCE` instead of `COMPLIANCE`. In GOVERNANCE mode, users with special permissions (`s3:BypassGovernanceRetention`) can override retention and delete protected objects. COMPLIANCE mode prevents any user, including root, from deleting or modifying objects during the retention period.

**Raw AWS CLI commands**

```bash
aws s3api get-object-lock-configuration --bucket phi-records
aws s3api get-bucket-tagging --bucket phi-records
```

**Bad AWS CLI output**

Object Lock enabled but in GOVERNANCE mode. Save as `lock.json` and `tagging.json`:

```bash
cat > lock.json <<'EOF'
{
  "ObjectLockConfiguration": {
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "GOVERNANCE",
        "Years": 7
      }
    }
  }
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "phi"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.data-classification eq "phi"`, `properties.storage.object_lock.enabled eq true`, and `properties.storage.object_lock.mode ne "COMPLIANCE"`.

```bash
mkdir -p observations

jq -n \
  --argjson lock "$(cat lock.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "phi-records",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "phi-records",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        object_lock: {
          enabled: (($lock.ObjectLockConfiguration.ObjectLockEnabled // "") == "Enabled"),
          mode: ($lock.ObjectLockConfiguration.Rule.DefaultRetention.Mode // null)
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```


**AWS doc basis for the fix**

```bash
aws s3api put-object-lock-configuration \
  --bucket phi-records \
  --object-lock-configuration '{
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Years": 7
      }
    }
  }'
```

In COMPLIANCE mode, no user (including the root account) can delete or modify protected objects during the retention period. This is required for HIPAA-regulated PHI data where tamper-proof storage is mandatory.
