## Anonymous S3 Listing Must Be Explicitly Intended

**Control:** CTL.S3.PUBLIC.LIST.002 (high)

**Misconfiguration:** The bucket allows public listing but has no intent tag confirming this is deliberate. Anonymous listing increases exposure surface even when objects are public by design — it should be an explicit choice, not an accident.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-policy --bucket public-site
aws s3api get-bucket-tagging --bucket public-site
```

**Bad AWS CLI output**

The bucket policy grants public listing, but no intent tag exists. Save the policy as `policy.json` and tags as `tagging.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicListBucket\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:ListBucket\",\"Resource\":\"arn:aws:s3:::public-site\"}]}"
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": []
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks three things:
- `properties.storage.kind eq "bucket"`
- `properties.storage.access.public_list eq true`
- `properties.storage.tags.public_list_intended` is missing or not `"true"`

```bash
mkdir -p observations

jq -n \
  --argjson policy "$(cat policy.json)" \
  --argjson tags "$(cat tagging.json)" \
'
  ($policy.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "public-site",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "public-site",
        access: {
          public_list: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:ListBucket" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:ListBucket" or . == "s3:*"))))
              )
            ] | length > 0
          )

  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

If anonymous listing is intentional, add the `public_list_intended` tag. Save as `tagging_fixed.json`:

```bash
cat > tagging_fixed.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "public_list_intended",
      "Value": "true"
    }
  ]
}
EOF
```

**AWS doc basis for the fix**

Either remove public `s3:ListBucket` grants if listing was accidental, or add the `public_list_intended=true` tag to confirm the listing is deliberate.
