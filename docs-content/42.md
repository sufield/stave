## CDN S3 Origins Must Not Be Dangling

**Control:** CTL.S3.DANGLING.ORIGIN.001 (high)

**Misconfiguration:** A CloudFront distribution has an S3 origin whose bucket name is not present in your owned buckets. The origin reference is dangling — an attacker can claim the bucket name and serve malicious content through your CDN domain.

**Raw AWS CLI commands**

```bash
aws s3api list-buckets
aws cloudfront list-distributions
```

**Bad AWS CLI output**

Your account owns two buckets but the CloudFront distribution references a third. Save as `buckets.json` and `distributions.json`:

```bash
cat > buckets.json <<'EOF'
{
  "Buckets": [
    { "Name": "app-assets-prod", "CreationDate": "2026-01-15T10:20:30+00:00" },
    { "Name": "media-prod", "CreationDate": "2026-01-18T09:12:10+00:00" }
  ]
}
EOF

cat > distributions.json <<'EOF'
{
  "DistributionList": {
    "Items": [
      {
        "Id": "E123EXAMPLEABC",
        "DomainName": "d111111abcdef8.cloudfront.net",
        "Origins": {
          "Items": [
            {
              "Id": "origin-1",
              "DomainName": "customer-downloads-prod.s3.us-east-1.amazonaws.com"
            }
          ]
        }
      }
    ]
  }
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.cdn.kind eq "distribution"` and `properties.cdn.origins_has_dangling_s3 eq true`. The extractor models the CloudFront distribution as the asset, checking whether any S3 origin references an unowned bucket.

```bash
mkdir -p observations

jq -n \
  --argjson buckets "$(cat buckets.json)" \
  --argjson dists "$(cat distributions.json)" \
'
  ($buckets.Buckets | map(.Name)) as $owned |
  ($dists.DistributionList.Items // [])[] |
  {
    schema_version: "obs.v0.1",
    generated_by: { source_type: "aws-cloudfront-snapshot", tool: "aws-cli", tool_version: "2.x" },
    captured_at: "2026-03-20T00:00:00Z",
    assets: [{
      id: .Id,
      type: "aws_cloudfront_distribution",
      vendor: "aws",
      properties: {
        cdn: {
          kind: "distribution",
          origins_has_dangling_s3: (
            [.Origins.Items[] |
              select(.DomainName | test("^[^.]+\\.s3[.-]")) |
              (.DomainName | capture("^(?<bucket>[^.]+)\\.s3[.-]").bucket) as $bucket |
              select(($owned | index($bucket)) == null)
            ] | length > 0
          )
        }
      }
    }]
  }
' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z \
  --allow-unknown-input
```

**Fixed AWS CLI output**

The missing bucket now exists in your account. Save as `buckets_fixed.json`:

```bash

  "Buckets": [
```

**AWS doc basis for the fix**

Create the referenced S3 bucket to claim the name, or update the CloudFront distribution to use an owned bucket with Origin Access Control (OAC):

```bash
# Option 1: Claim the bucket
aws s3api create-bucket --bucket customer-downloads-prod --region us-east-1

# Option 2: Update origin to an owned bucket
aws cloudfront get-distribution-config --id E123EXAMPLEABC
# Edit config to point origin to media-prod.s3.us-east-1.amazonaws.com
aws cloudfront update-distribution --id E123EXAMPLEABC --distribution-config file://config.json --if-match ETAG
```
