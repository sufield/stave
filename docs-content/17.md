## No Public ACL Readability

**Control:** CTL.S3.ACL.RECON.001 (high)

**Misconfiguration:** The bucket ACL grants `READ_ACP` to the `AllUsers` group, so anyone can read the bucket's ACL and learn who has access. This enables attackers to enumerate permissions and find escalation paths.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-acl --bucket recon-target
```

**Bad AWS CLI output**

Save as `acl.json`:

```bash
cat > acl.json <<'EOF'
{
  "Owner": {
    "DisplayName": "bucket-owner",
    "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
  },
  "Grants": [
    {
      "Grantee": {
        "Type": "CanonicalUser",
        "DisplayName": "bucket-owner",
        "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
      },
      "Permission": "FULL_CONTROL"
    },
    {
      "Grantee": {
        "Type": "Group",
        "URI": "http://acs.amazonaws.com/groups/global/AllUsers"
      },
      "Permission": "READ_ACP"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.public_admin`. The extractor checks whether any ACL grant gives `READ_ACP`, `WRITE_ACP`, or `FULL_CONTROL` to the `AllUsers` group — all of which expose administrative ACL operations publicly. Any of these constitute public ACL-based access.

```bash
mkdir -p observations

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "recon-target",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "recon-target",
        access: {
          public_admin: (
            [.Grants[] |
              select(
                (.Permission == "READ_ACP" or .Permission == "WRITE_ACP" or .Permission == "FULL_CONTROL")
                and .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AllUsers"))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "recon-target",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "recon-target",
        access: {
          public_admin: (
            [.Grants[] |
              select(
                (.Permission == "READ_ACP" or .Permission == "WRITE_ACP" or .Permission == "FULL_CONTROL")
                and .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AllUsers"))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**AWS doc basis for the fix**

Remove the `AllUsers` `READ_ACP` grant from the bucket ACL. For modern S3, use **Bucket owner enforced** object ownership to disable ACLs entirely.
