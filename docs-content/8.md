## Complete Data Required for Safety Assessment

**Control:** CTL.S3.INCOMPLETE.001 (low)

**Misconfiguration:** The observation snapshot is missing required evidence, so Stave cannot determine whether the bucket is safe or unsafe. This happens when the extractor fails to call all required AWS APIs (e.g., `get-public-access-block`, `get-bucket-policy-status`, `get-bucket-versioning`, `get-bucket-encryption`, `get-bucket-logging`, `get-bucket-tagging`) or lacks the IAM permissions to read them.

The control checks a single field: `properties.safety_provable`. The extractor sets this to `false` when it cannot gather complete evidence, and `true` when all required API calls succeed.

**Raw AWS CLI commands**

```bash
aws s3api get-public-access-block --bucket audit-trail
aws s3api get-bucket-policy-status --bucket audit-trail
aws s3api get-bucket-versioning --bucket audit-trail
aws s3api get-bucket-encryption --bucket audit-trail
aws s3api get-bucket-logging --bucket audit-trail
aws s3api get-bucket-tagging --bucket audit-trail
```

**Bad extractor output (incomplete evidence)**

When some API calls fail (e.g., due to missing IAM permissions), the extractor marks the observation as not provable. Save as `incomplete.json`:

```bash
cat > incomplete.json <<'EOF'
{
  "safety_provable": false,
  "partial_data": {
    "versioning": {"Status": "Enabled"},
    "encryption": {"Rules": [{"SSEAlgorithm": "AES256"}]}
  },
  "missing": ["public_access_block", "policy_status", "logging", "tagging"]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control only checks `properties.safety_provable`. The extractor passes through the provability flag from its own collection status.

**Fix**

Ensure the extractor always calls every required AWS CLI command for the bucket. If any call fails, set `safety_provable: false`. The required IAM permissions are:
- `s3:GetBucketPolicy`
- `s3:GetBucketAcl`
- `s3:GetBucketPolicyStatus`
- `s3:GetPublicAccessBlock`
- `s3:GetBucketVersioning`
- `s3:GetEncryptionConfiguration`
- `s3:GetBucketLogging`
- `s3:GetBucketTagging`

The problem is not a single bad S3 setting. The problem is **missing evidence**.
