## Retention-Tagged Buckets Must Have Lifecycle Rules

**Control:** CTL.S3.LIFECYCLE.001 (medium)

**Misconfiguration:** The bucket is tagged with `data-retention` but has no lifecycle configuration, so objects accumulate without automatic expiration or transition. HIPAA requires defined data retention policies for PHI, audit logs, and billing records.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-lifecycle-configuration --bucket compliance-logs
aws s3api get-bucket-tagging --bucket compliance-logs
```

**Bad AWS CLI output**

No lifecycle rules, but a retention tag exists. Save as `lifecycle.json` and `tagging.json`:

```bash
cat > lifecycle.json <<'EOF'
{
  "Rules": []
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-retention",
      "Value": "7-years"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.data-retention` (present) AND `properties.storage.lifecycle.rules_configured eq false`.

```bash
mkdir -p observations

jq -n \
  --argjson lc "$(cat lifecycle.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "compliance-logs",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "compliance-logs",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        lifecycle: {
          rules_configured: (($lc.Rules // []) | length > 0)
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

A lifecycle rule for 365-day expiration. Save as `lifecycle_fixed.json`:

```bash
cat > lifecycle_fixed.json <<'EOF'
{
  "Rules": [
    {
      "ID": "expire-after-365-days",
      "Status": "Enabled",
      "Filter": { "Prefix": "" },
      "Expiration": { "Days": 365 }
    }
  ]
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

jq -n \
  --argjson lc "$(cat lifecycle_fixed.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "compliance-logs",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "compliance-logs",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        lifecycle: {
          rules_configured: (($lc.Rules // []) | length > 0)
        }
      }
    }
  }]
}' > observations-after/2026-03-22T000000Z.json

stave apply \
  --observations observations-after \
  --max-unsafe 12h \
  --now 2026-03-22T12:00:00Z
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-lifecycle-configuration \
  --bucket compliance-logs \
  --lifecycle-configuration '{
    "Rules": [
      {
        "ID": "expire-after-365-days",
        "Status": "Enabled",
        "Filter": { "Prefix": "" },
        "Expiration": { "Days": 365 }
      }
    ]
  }'
```
