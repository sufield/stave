## PHI Object Lock Retention Must Meet Minimum Period

**Control:** CTL.S3.LOCK.003 (medium)

**Misconfiguration:** The bucket is tagged `data-classification: phi` and has Object Lock enabled, but the default retention period is only 365 days — far below the HIPAA minimum of 6 years (2190 days). Short retention periods mean WORM protection expires before the regulatory retention obligation, allowing deletion of PHI data.

**Raw AWS CLI commands**

```bash
aws s3api get-object-lock-configuration --bucket phi-vault
aws s3api get-bucket-tagging --bucket phi-vault
```

**Bad AWS CLI output**

Object Lock with only 365-day retention on a PHI bucket. Save as `lock.json` and `tagging.json`:

```bash
cat > lock.json <<'EOF'
{
  "ObjectLockConfiguration": {
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Days": 365
      }
    }
  }
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "phi"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.data-classification eq "phi"`, `properties.storage.object_lock.enabled eq true`, and `properties.storage.object_lock.retention_days lt 2190`.

```bash
mkdir -p observations

jq -n \
  --argjson lock "$(cat lock.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "phi-vault",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "phi-vault",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        object_lock: {
          enabled: (($lock.ObjectLockConfiguration.ObjectLockEnabled // "") == "Enabled"),
          retention_days: (
            if $lock.ObjectLockConfiguration.Rule.DefaultRetention.Days != null
            then $lock.ObjectLockConfiguration.Rule.DefaultRetention.Days
            elif $lock.ObjectLockConfiguration.Rule.DefaultRetention.Years != null
            then $lock.ObjectLockConfiguration.Rule.DefaultRetention.Years * 365
            else 0 end
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Retention increased to 2555 days (~7 years). Save as `lock_fixed.json`:

```bash
cat > lock_fixed.json <<'EOF'
{
  "ObjectLockConfiguration": {
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Days": 2555
      }
    }
  }
}
EOF
```

**AWS doc basis for the fix**

```bash
aws s3api put-object-lock-configuration \
  --bucket phi-vault \
  --object-lock-configuration '{
    "ObjectLockEnabled": "Enabled",
    "Rule": {
      "DefaultRetention": {
        "Mode": "COMPLIANCE",
        "Days": 2555
      }
    }
  }'
```

The default threshold is 2190 days (6 years, HIPAA minimum). AWS Object Lock supports retention in either `Days` or `Years`. When using `Years`, multiply by 365 to get the equivalent days for comparison.
