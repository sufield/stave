## No Public S3 Buckets With Sensitive Data

**Control:** CTL.S3.PUBLIC.002 (critical)

**Misconfiguration:** The bucket is tagged `data-classification: confidential` but also allows public read, so sensitive data is exposed to the internet. The sensitivity label contradicts the actual access.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-policy --bucket patient-records
aws s3api get-bucket-tagging --bucket patient-records
```

**Bad AWS CLI output**

A public-read policy exists on a confidential bucket. Save as `policy.json` and `tagging.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadGetObject\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::patient-records/*\"}]}"
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "confidential"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.public_read` AND `properties.storage.tags.data-classification` being `phi`, `pii`, or `confidential`. The extractor derives `public_read` from the policy and maps tags from the TagSet.

```bash
mkdir -p observations

jq -n \
  --argjson policy "$(cat policy.json)" \
  --argjson tags "$(cat tagging.json)" \
'
  ($policy.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "patient-records",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "patient-records",
        access: {
          public_read: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:GetObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:GetObject" or . == "s3:*"))))
              )
            ] | length > 0
          )
        },
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {})
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

Remove the public read policy. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppReadRole\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::patient-records/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the public-read statement from the bucket policy. Sensitive data buckets (PHI, PII, confidential) should never have public access. Enable S3 Public Access Block as defense-in-depth.
