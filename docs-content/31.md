## PHI Buckets Must Not Expire Data Before Minimum Retention

**Control:** CTL.S3.LIFECYCLE.002 (medium)

**Misconfiguration:** The bucket is tagged `data-classification: phi` but its lifecycle rules expire objects after only 30 days — far below the HIPAA minimum retention of 6 years (2190 days). Premature deletion of PHI violates retention requirements.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-lifecycle-configuration --bucket phi-archive
aws s3api get-bucket-tagging --bucket phi-archive
```

**Bad AWS CLI output**

A 30-day expiration on a PHI bucket. Save as `lifecycle.json` and `tagging.json`:

```bash
cat > lifecycle.json <<'EOF'
{
  "Rules": [
    {
      "ID": "expire-phi-too-early",
      "Status": "Enabled",
      "Filter": { "Prefix": "" },
      "Expiration": { "Days": 30 }
    }
  ]
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "phi"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.data-classification eq "phi"`, `properties.storage.lifecycle.has_expiration eq true`, and `properties.storage.lifecycle.min_expiration_days lt 2190`.

```bash
mkdir -p observations

jq -n \
  --argjson lc "$(cat lifecycle.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "phi-archive",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "phi-archive",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        lifecycle: {
          has_expiration: (
            [($lc.Rules // [])[] | select(.Status == "Enabled" and .Expiration.Days != null)] | length > 0
          ),
          min_expiration_days: (
            [($lc.Rules // [])[] | select(.Status == "Enabled" and .Expiration.Days != null) | .Expiration.Days] |
            if length == 0 then 0 else min end
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Expiration increased to 2555 days (~7 years). Save as `lifecycle_fixed.json`:

```bash
cat > lifecycle_fixed.json <<'EOF'
{
  "Rules": [
    {
      "ID": "expire-phi-after-minimum-retention",
      "Status": "Enabled",
      "Filter": { "Prefix": "" },
      "Expiration": { "Days": 2555 }
    }

}
EOF
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-lifecycle-configuration \
  --bucket phi-archive \
  --lifecycle-configuration '{
    "Rules": [
      {
        "ID": "expire-phi-after-minimum-retention",
        "Status": "Enabled",
        "Filter": { "Prefix": "" },
        "Expiration": { "Days": 2555 }
      }
    ]
  }'
```

The default threshold is 2190 days (6 years, HIPAA minimum). The threshold is configurable via `params.min_retention_days` in the control definition — copy the control to your `controls/` directory and change the value in the `params` section to match your retention policy.
