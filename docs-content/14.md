## Public-Principal Policies Must Have Network Conditions

**Control:** CTL.S3.NETWORK.001 (high)

**Misconfiguration:** A bucket policy has an `Allow` statement with `Principal: "*"` but no network restriction such as `aws:SourceVpc`, `aws:SourceVpce`, or `aws:SourceIp`, so access is effectively internet-wide.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket cdn-origin
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadNoNetworkFence\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::cdn-origin/*\"}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.effective_network_scope`. The extractor parses the policy and determines the network scope: `"public"` when a wildcard-principal statement has no network conditions, or `"restricted"` when conditions are present.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
  ([$p.Statement[] |
    select(
      .Effect == "Allow"
      and (.Principal == "*" or .Principal.AWS == "*")
    )
  ]) as $public_stmts |
  (if ($public_stmts | length) == 0 then "none"
   elif ($public_stmts | any(
     (.Condition // {}) as $c |
     ($c.StringEquals["aws:SourceVpc"] // null) == null
     and ($c.StringEquals["aws:SourceVpce"] // null) == null
     and ($c.IpAddress["aws:SourceIp"] // null) == null
   )) then "public"
   else "restricted"
   end) as $scope |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "cdn-origin",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "cdn-origin",
        access: {
          effective_network_scope: $scope
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

A source IP condition restricts access to a corporate IP range. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadRestrictedToCorpIP\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::cdn-origin/*\",\"Condition\":{\"IpAddress\":{\"aws:SourceIp\":\"203.0.113.0/24\"}}}]}"
}
EOF
```

**AWS doc basis for the fix**

Add a network condition to the public-principal `Allow` statement, or remove the public principal entirely. Options:
- `aws:SourceIp` for IP range restrictions
- `aws:SourceVpc` for VPC restrictions
- `aws:SourceVpce` for VPC endpoint restrictions
