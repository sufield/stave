## Backup Buckets Must Have MFA Delete Enabled

**Control:** CTL.S3.VERSION.002 (medium)

**Misconfiguration:** A backup bucket has versioning enabled but MFA Delete is not enabled, so someone with stolen long-term credentials could permanently delete object versions or change the bucket's versioning state without an MFA code. AWS says MFA Delete adds MFA protection to permanently deleting object versions and changing versioning state.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-versioning --bucket backup-vault
```

**Bad AWS CLI output**

AWS documents that `GetBucketVersioning` returns `Status`, and returns `MFADelete` only if MFA Delete has been configured. A versioned bucket with no `MFADelete` field is the "not enabled" case. Save as `versioning.json`:

```bash
cat > versioning.json <<'EOF'
{
  "Status": "Enabled"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.backup` and `properties.storage.versioning.mfa_delete_enabled`. The extractor derives `mfa_delete_enabled` from whether `MFADelete` equals `"Enabled"`.

```bash
mkdir -p observations

cat versioning.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "backup-vault",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "backup-vault",
        tags: {
          backup: "true"
        },
        versioning: {
          enabled: (.Status == "Enabled"),
          mfa_delete_enabled: (.MFADelete == "Enabled")
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat versioning.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
```

**Fixed AWS CLI output**

Save as `versioning_fixed.json`:

```bash
cat > versioning_fixed.json <<'EOF'
{
  "MFADelete": "Enabled",
  "Status": "Enabled"
}
EOF
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-versioning \
  --bucket backup-vault \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "<mfa-device-serial> <mfa-code>"
```

Only the bucket owner (root account) can enable MFA Delete. AWS also notes that MFA Delete cannot be used with lifecycle configurations on the bucket.
