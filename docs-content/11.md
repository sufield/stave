## No Public Write Access

**Control:** CTL.S3.PUBLIC.003 (critical)

**Misconfiguration:** The bucket policy grants public write actions such as `s3:PutObject` or `s3:DeleteObject` to `Principal: "*"`, which lets anyone upload, replace, or delete objects. This can enable data injection, defacement, or destructive deletion.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket uploads
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicWrite\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":[\"s3:PutObject\",\"s3:DeleteObject\"],\"Resource\":\"arn:aws:s3:::uploads/*\"}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.public_write`. The extractor parses the bucket policy and checks whether any `Allow` statement grants write actions to `Principal: "*"`.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "uploads",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "uploads",
        access: {
          public_write: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (
                  .Action == "s3:PutObject" or .Action == "s3:DeleteObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:PutObject" or . == "s3:DeleteObject" or . == "s3:*")))
                )
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

The public write statement is replaced with access limited to a specific IAM role. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppWrite\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppWriteRole\"},\"Action\":[\"s3:PutObject\",\"s3:DeleteObject\"],\"Resource\":\"arn:aws:s3:::uploads/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove any statement that grants `s3:PutObject`, `s3:DeleteObject`, or `s3:*` to `Principal: "*"`. Replace with access granted only to specific AWS principals. Use `get-bucket-policy-status` to verify the bucket is no longer considered public.
