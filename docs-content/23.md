## No Latent Public Read Exposure

**Control:** CTL.S3.PUBLIC.005 (medium)

**Misconfiguration:** The bucket has a public-read policy underneath, but Block Public Access is currently masking it. The bucket is not publicly readable right now, yet it becomes public immediately if Block Public Access is removed. This is a fragile state — safety depends on a single override rather than the policy itself being correct.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-policy --bucket staging-data
aws s3api get-public-access-block --bucket staging-data
```

**Bad AWS CLI output**

A public-read policy exists, but Block Public Access masks it. Save as `policy.json` and `pab.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadGetObject\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::staging-data/*\"}]}"
}
EOF

cat > pab.json <<'EOF'
{
  "PublicAccessBlockConfiguration": {
    "BlockPublicAcls": true,
    "IgnorePublicAcls": true,
    "BlockPublicPolicy": true,
    "RestrictPublicBuckets": true
  }
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.latent_public_read`. The extractor derives this: `true` when a public-read policy exists AND Block Public Access is the only barrier.

```bash
mkdir -p observations

jq -n \
  --argjson policy "$(cat policy.json)" \
  --argjson pab "$(cat pab.json)" \
'
  ($policy.Policy | fromjson) as $p |
  ([$p.Statement[] |
    select(
      .Effect == "Allow"
      and (.Principal == "*" or .Principal.AWS == "*")
      and (.Action == "s3:GetObject" or .Action == "s3:*"
        or ((.Action | type) == "array"
          and (.Action | any(. == "s3:GetObject" or . == "s3:*"))))
    )
  ] | length > 0) as $has_public_read |
  ($pab.PublicAccessBlockConfiguration |
    (.BlockPublicPolicy == true) or (.RestrictPublicBuckets == true)
  ) as $pab_blocking |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "staging-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "staging-data",
        access: {
          latent_public_read: ($has_public_read and $pab_blocking)
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

Remove the public policy; keep Block Public Access as defense-in-depth. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppReadRole\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::staging-data/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the underlying public-read statement from the bucket policy. Keep Block Public Access enabled as defense-in-depth rather than as the sole barrier.
