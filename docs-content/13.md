## No External Write Access

**Control:** CTL.S3.ACCESS.003 (high)

**Misconfiguration:** The bucket policy grants write access to an external AWS account, so a principal outside the bucket owner's account can upload, overwrite, or delete objects. Cross-account read may be acceptable for analytics, but write access from external accounts creates data integrity and supply chain risks.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket internal-data
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"ExternalWriteAccess\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::999988887777:root\"},\"Action\":[\"s3:PutObject\",\"s3:DeleteObject\"],\"Resource\":\"arn:aws:s3:::internal-data/*\"}]}"
}
EOF
```

The principal `999988887777` is a different account from the bucket owner (`123456789012`).

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.has_external_write`. The extractor parses the policy and checks whether any `Allow` statement grants write actions to a principal from a different AWS account.

Replace `123456789012` with your own account ID.

```bash
mkdir -p observations

OWNER_ACCOUNT="123456789012"

cat policy.json | jq --arg owner "$OWNER_ACCOUNT" '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "internal-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "internal-data",
        access: {
          has_external_write: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal.AWS != null and .Principal.AWS != "*")
                and ((.Principal.AWS | tostring) | contains($owner) | not)
                and (
                  .Action == "s3:PutObject" or .Action == "s3:DeleteObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:PutObject" or . == "s3:DeleteObject" or . == "s3:*")))
                )
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat policy.json | jq --arg owner "$OWNER_ACCOUNT" '
```

**Fixed AWS CLI output**

The external account now has read-only access instead of write. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"ExternalReadOnlyAccess\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::999988887777:root\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::internal-data/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the statement that grants `s3:PutObject`, `s3:DeleteObject`, or other write actions to the external account, or replace with a read-only statement.
