## Encryption at Rest Required

**Control:** CTL.S3.ENCRYPT.001 (high)

**Misconfiguration:** The bucket does not have server-side encryption enabled. Unencrypted storage is the top audit finding in regulated industries. This control checks whether encryption at rest is explicitly configured.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-encryption --bucket data-lake
```

**Bad AWS CLI output (no encryption configured)**

When a bucket has no default encryption, the AWS CLI returns an error. Save the absence as `encryption.json`:

```bash
cat > encryption.json <<'EOF'
{
  "ServerSideEncryptionConfiguration": {
    "Rules": []
  }
}
EOF
```

An empty `Rules` array means no default encryption is configured.

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind` and `properties.storage.encryption.at_rest_enabled`. The extractor derives `at_rest_enabled` from whether any encryption rule exists.

```bash
mkdir -p observations

cat encryption.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "data-lake",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "data-lake",
        encryption: {
          at_rest_enabled: (
            (.ServerSideEncryptionConfiguration.Rules | length) > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat encryption.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "data-lake",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "data-lake",
        encryption: {
          at_rest_enabled: (
            (.ServerSideEncryptionConfiguration.Rules | length) > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

After enabling SSE-KMS encryption, save as `encryption_fixed.json`:

```bash
cat > encryption_fixed.json <<'EOF'
{
  "ServerSideEncryptionConfiguration": {
    "Rules": [
      {
        "ApplyServerSideEncryptionByDefault": {
          "SSEAlgorithm": "aws:kms",
          "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789012:key/11111111-2222-3333-4444-555555555555"
        },
        "BucketKeyEnabled": true
      }
    ]
  }
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

cat encryption_fixed.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "data-lake",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "data-lake",
        encryption: {
          at_rest_enabled: (

          )
        }

}' > observations-after/2026-03-22T000000Z.json

stave apply \

```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-encryption \
  --bucket data-lake \
  --server-side-encryption-configuration '{
    "Rules": [{
      "ApplyServerSideEncryptionByDefault": {
        "SSEAlgorithm": "aws:kms",
        "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789012:key/11111111-2222-3333-4444-555555555555"
      },
      "BucketKeyEnabled": true
    }]
  }'
```
