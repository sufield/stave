## Access Logging Required

**Control:** CTL.S3.LOG.001 (medium)

**Misconfiguration:** Server access logging is not enabled for the bucket, so there is no audit trail of requests made to that bucket. AWS says server access logging provides detailed records for requests to a bucket and is useful for security and access audits.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-logging --bucket app-data
```

**Bad AWS CLI output**

When logging is not configured, `get-bucket-logging` returns an empty object. Save as `logging.json`:

```bash
cat > logging.json <<'EOF'
{}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind` and `properties.storage.logging.enabled`. The extractor derives `enabled` from whether `LoggingEnabled` is present in the response.

```bash
mkdir -p observations

cat logging.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "app-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "app-data",
        logging: {
          enabled: has("LoggingEnabled")
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat logging.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "app-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "app-data",
        logging: {
          enabled: has("LoggingEnabled")
        }
      }
    }
  }]
}' > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Save as `logging_fixed.json`:

```bash
cat > logging_fixed.json <<'EOF'
{
  "LoggingEnabled": {
    "TargetBucket": "central-s3-access-logs",
    "TargetPrefix": "app-data/"
  }
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

cat logging_fixed.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "app-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "app-data",
        logging: {
          enabled: has("LoggingEnabled")
        }
      }
    }
  }]
}' > observations-after/2026-03-22T000000Z.json

stave apply \
  --observations observations-after \
  --max-unsafe 12h \
  --now 2026-03-22T12:00:00Z
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-logging \
  --bucket app-data \
  --bucket-logging-status '{
    "LoggingEnabled": {
      "TargetBucket": "central-s3-access-logs",
      "TargetPrefix": "app-data/"
    }
  }'
```
