## No Public List via Policy

**Control:** CTL.S3.PUBLIC.008 (critical)

**Misconfiguration:** The bucket policy grants `s3:ListBucket` to `Principal: "*"`, allowing anyone on the internet to enumerate all objects in the bucket. Object listing reveals file names, sizes, and timestamps — valuable reconnaissance data for an attacker.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket internal-docs
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicListBucket\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:ListBucket\",\"Resource\":\"arn:aws:s3:::internal-docs\"}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.list_via_identity eq true`. The extractor parses the bucket policy and checks whether any `Allow` statement grants listing to `Principal: "*"`.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "internal-docs",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "internal-docs",
        access: {
          list_via_identity: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:ListBucket" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:ListBucket" or . == "s3:*"))))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

The public listing statement is replaced with access limited to a specific IAM role. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppList\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppListRole\"},\"Action\":\"s3:ListBucket\",\"Resource\":\"arn:aws:s3:::internal-docs\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the statement that grants `s3:ListBucket` to `Principal: "*"`, or replace it with a statement that grants access only to a specific AWS principal.

```bash
aws s3api get-bucket-policy --bucket internal-docs --query Policy --output text > policy.json
# Edit policy.json to remove public listing statement
aws s3api put-bucket-policy --bucket internal-docs --policy file://policy.json
```
