## No Latent Public Bucket Listing

**Control:** CTL.S3.PUBLIC.006 (critical)

**Misconfiguration:** The bucket has a public listing grant underneath, but Block Public Access is currently masking it. The bucket is not publicly listable right now, but it becomes publicly listable immediately if Block Public Access is removed. This is one configuration change away from exposing all object keys.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-policy --bucket project-artifacts
aws s3api get-public-access-block --bucket project-artifacts
```

**Bad AWS CLI output**

A public `ListBucket` policy exists, but Block Public Access masks it. Save as `policy.json` and `pab.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicListBucket\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:ListBucket\",\"Resource\":\"arn:aws:s3:::project-artifacts\"}]}"
}
EOF

cat > pab.json <<'EOF'
{
  "PublicAccessBlockConfiguration": {
    "BlockPublicAcls": true,
    "IgnorePublicAcls": true,
    "BlockPublicPolicy": true,
    "RestrictPublicBuckets": true
  }
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind` and `properties.storage.access.latent_public_list`. The extractor derives `latent_public_list`: `true` when a public listing policy exists AND Block Public Access is the only barrier.

```bash
mkdir -p observations

jq -n \
  --argjson policy "$(cat policy.json)" \
  --argjson pab "$(cat pab.json)" \
'
  ($policy.Policy | fromjson) as $p |
  ([$p.Statement[] |
    select(
      .Effect == "Allow"
      and (.Principal == "*" or .Principal.AWS == "*")
      and (.Action == "s3:ListBucket" or .Action == "s3:*"
        or ((.Action | type) == "array"
          and (.Action | any(. == "s3:ListBucket" or . == "s3:*"))))
    )
  ] | length > 0) as $has_public_list |
  ($pab.PublicAccessBlockConfiguration |
    (.BlockPublicPolicy == true) or (.RestrictPublicBuckets == true)
  ) as $pab_blocking |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "project-artifacts",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "project-artifacts",
        access: {
          latent_public_list: ($has_public_list and $pab_blocking)
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Fixed AWS CLI output**

Remove the public listing policy; keep Block Public Access as defense-in-depth. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppReadOnly\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppReadRole\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::project-artifacts/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the underlying public `s3:ListBucket` statement from the bucket policy. Keep Block Public Access enabled as defense-in-depth.
