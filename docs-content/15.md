## No Public Read via ACL

**Control:** CTL.S3.PUBLIC.004 (medium)

**Misconfiguration:** The bucket ACL grants `READ` to the `AllUsers` group, which makes bucket contents publicly readable without authentication. AWS documents the global `AllUsers` group URI as `http://acs.amazonaws.com/groups/global/AllUsers`.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-acl --bucket public-assets
```

**Bad AWS CLI output**

Save as `acl.json`:

```bash
cat > acl.json <<'EOF'
{
  "Owner": {
    "DisplayName": "bucket-owner",
    "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
  },
  "Grants": [
    {
      "Grantee": {
        "Type": "CanonicalUser",
        "DisplayName": "bucket-owner",
        "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
      },
      "Permission": "FULL_CONTROL"
    },
    {
      "Grantee": {
        "Type": "Group",
        "URI": "http://acs.amazonaws.com/groups/global/AllUsers"
      },
      "Permission": "READ"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.read_via_resource`. The extractor checks whether any ACL grant gives `READ` or `FULL_CONTROL` to the `AllUsers` or `AuthenticatedUsers` groups.

```bash
mkdir -p observations

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "public-assets",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "public-assets",
        access: {
          read_via_resource: (
            [.Grants[] |
              select(
                .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AllUsers") or contains("AuthenticatedUsers"))
                and (.Permission == "READ" or .Permission == "FULL_CONTROL")
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "public-assets",
    type: "aws_s3_bucket",

    properties: {

```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

The `AllUsers` grant is removed, leaving only the owner's control grant. Save as `acl_fixed.json`:


**AWS doc basis for the fix**

Remove the `AllUsers` `READ` grant from the bucket ACL, or reset to private. Modern S3 buckets can use **Bucket owner enforced** object ownership to disable ACLs entirely.
