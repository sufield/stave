## No Public Website Hosting with Public Read

**Control:** CTL.S3.WEBSITE.PUBLIC.001 (critical)

**Misconfiguration:** The bucket has S3 static website hosting enabled and also allows public read access, serving content directly to the internet via the S3 website endpoint. This combination is flagged because website hosting with public read exposes all objects without CloudFront protections (WAF, geo-restrictions, HTTPS).

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-website --bucket marketing-site
aws s3api get-bucket-policy --bucket marketing-site
```

**Bad AWS CLI output**

Website hosting enabled with a public-read policy. Save as `website.json` and `policy.json`:

```bash
cat > website.json <<'EOF'
{
  "IndexDocument": {
    "Suffix": "index.html"
  },
  "ErrorDocument": {
    "Key": "error.html"
  }
}
EOF

cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadForWebsite\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::marketing-site/*\"}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.website.enabled eq true` and `properties.storage.access.public_read eq true`.

```bash
mkdir -p observations

jq -n \
  --argjson website "$(cat website.json)" \
  --argjson policy "$(cat policy.json | jq '.Policy | fromjson')" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "marketing-site",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "marketing-site",
        website: {
          enabled: ($website.IndexDocument != null)
        },
        access: {
          public_read: (
            [$policy.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:GetObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:GetObject" or . == "s3:*"))))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Website hosting disabled and public read replaced with CloudFront OAC access. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowCloudFrontOAC\",\"Effect\":\"Allow\",\"Principal\":{\"Service\":\"cloudfront.amazonaws.com\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::marketing-site/*\",\"Condition\":{\"StringEquals\":{\"AWS:SourceArn\":\"arn:aws:cloudfront::123456789012:distribution/E2ABCDEFGHIJKL\"}}}]}"
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

jq -n \
  --argjson policy "$(cat policy_fixed.json | jq '.Policy | fromjson')" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "marketing-site",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "marketing-site",
        website: {
          enabled: false
        },
        access: {
          public_read: (
            [$policy.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:GetObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:GetObject" or . == "s3:*"))))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations-after/2026-03-22T000000Z.json

stave apply \
  --observations observations-after \
  --max-unsafe 12h \
  --now 2026-03-22T12:00:00Z
```

**AWS doc basis for the fix**

Disable S3 website hosting and put CloudFront in front of a private S3 origin with Origin Access Control (OAC). S3 website endpoints do not support HTTPS, so CloudFront also adds TLS termination.

```bash
aws s3api delete-bucket-website --bucket marketing-site
aws s3api put-bucket-policy --bucket marketing-site --policy file://policy_fixed.json
```
