## Referenced S3 Buckets Must Exist And Be Owned

**Control:** CTL.S3.BUCKET.TAKEOVER.001 (critical)

**Misconfiguration:** A CloudFront distribution references an S3 bucket that does not exist in your account. An attacker can claim the bucket name and serve malicious content through your CDN domain. This is a bucket takeover vulnerability.

**Raw AWS CLI commands**

```bash
aws s3api list-buckets
aws cloudfront list-distributions
```

**Bad AWS CLI output**

Your account owns two buckets but CloudFront references a third that doesn't exist. Save as `buckets.json` and `distributions.json`:

```bash
cat > buckets.json <<'EOF'
{
  "Buckets": [
    { "Name": "app-assets-prod", "CreationDate": "2026-01-15T10:20:30+00:00" },
    { "Name": "media-prod", "CreationDate": "2026-01-18T09:12:10+00:00" }
  ]
}
EOF

cat > distributions.json <<'EOF'
{
  "DistributionList": {
    "Items": [
      {
        "Id": "E123EXAMPLEABC",
        "DomainName": "d111111abcdef8.cloudfront.net",
        "Origins": {
          "Items": [
            {
              "Id": "missing-origin",
              "DomainName": "customer-downloads-prod.s3.us-east-1.amazonaws.com"
            }
          ]
        }
      }
    ]
  }
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.s3_ref.bucket_exists eq false` or `properties.s3_ref.bucket_owned eq false`. The extractor cross-references CloudFront S3 origins against owned buckets.

```bash
mkdir -p observations

jq -n \
  --argjson buckets "$(cat buckets.json)" \
  --argjson cfdists "$(cat distributions.json)" \
'
  ($buckets.Buckets | map(.Name)) as $owned |
  [
    ($cfdists.DistributionList.Items // [])[] |
    .Origins.Items[] |
    select(.DomainName | test("^[^.]+\\.s3[.-]")) |
    (.DomainName | capture("^(?<bucket>[^.]+)\\.s3[.-]").bucket) as $bucket |
    {
      schema_version: "obs.v0.1",
      generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
      captured_at: "2026-03-20T00:00:00Z",
      assets: [{
        id: $bucket,
        type: "aws_s3_bucket",
        vendor: "aws",
        properties: {
          s3_ref: {
            bucket_exists: ($owned | index($bucket) != null),
            bucket_owned: ($owned | index($bucket) != null)
          }
        }
      }]
    }
  ] | .[0]
' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

The referenced bucket now exists in your account. Save as `buckets_fixed.json`:

```bash
cat > buckets_fixed.json <<'EOF'
{
  "Buckets": [
    { "Name": "app-assets-prod", "CreationDate": "2026-01-15T10:20:30+00:00" },

    { "Name": "customer-downloads-prod", "CreationDate": "2026-03-23T14:00:00+00:00" }

}
EOF
```

**AWS doc basis for the fix**

Create the referenced S3 bucket to claim the name, or remove the dangling origin from the CloudFront distribution:

```bash
# Option 1: Claim the bucket
aws s3api create-bucket --bucket customer-downloads-prod --region us-east-1

# Option 2: Remove the dangling origin
aws cloudfront get-distribution-config --id E123EXAMPLEABC
# Edit the config to remove the dangling origin, then update
aws cloudfront update-distribution --id E123EXAMPLEABC --distribution-config file://config.json --if-match ETAG
```
