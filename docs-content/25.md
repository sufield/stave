## No Unauthorized Cross-Account Access

**Control:** CTL.S3.ACCESS.001 (high)

**Misconfiguration:** The bucket policy grants access to an AWS account that is not in the approved allowlist. Cross-account access via bucket policy is expressed through the `Principal` element, which can name another AWS account, role, or user.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket compliance-reports
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"UnauthorizedExternalRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::999988887777:root\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::compliance-reports/*\"}]}"
}
EOF
```

Account `999988887777` is not in the approved allowlist.

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.external_account_ids` against `params.allowed_accounts` (defaults to empty — no external access allowed). The extractor parses the policy and collects external account IDs from `Principal.AWS` fields.

```bash
mkdir -p observations

OWNER_ACCOUNT="123456789012"

cat policy.json | jq --arg owner "$OWNER_ACCOUNT" '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "compliance-reports",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "compliance-reports",
        access: {
          external_account_ids: [
            $p.Statement[] |
            select(.Effect == "Allow") |
            .Principal.AWS |
            if type == "array" then .[] else . end |
            select(type == "string" and . != "*") |
            capture("arn:aws:iam::(?<acct>[0-9]{12}):") |
            .acct |
            select(. != $owner)
          ]
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat policy.json | jq --arg owner "$OWNER_ACCOUNT" '
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

The unauthorized external account is replaced with the owner account. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AuthorizedRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:root\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::compliance-reports/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove statements granting access to unauthorized external accounts. Use `aws:PrincipalOrgID` condition to restrict access to your AWS Organization.
