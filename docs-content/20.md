## No Authenticated-Users Write Access

**Control:** CTL.S3.AUTH.WRITE.001 (high)

**Misconfiguration:** The bucket ACL grants `WRITE` to the `AuthenticatedUsers` group, which does **not** mean "our internal AWS users." It means any AWS account that can send an authenticated request. On a bucket ACL, `WRITE` allows creating, overwriting, and deleting objects.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-acl --bucket ingest-pipeline
```

**Bad AWS CLI output**

Save as `acl.json`:

```bash
cat > acl.json <<'EOF'
{
  "Owner": {
    "DisplayName": "bucket-owner",
    "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
  },
  "Grants": [
    {
      "Grantee": {
        "Type": "CanonicalUser",
        "DisplayName": "bucket-owner",
        "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
      },
      "Permission": "FULL_CONTROL"
    },
    {
      "Grantee": {
        "Type": "Group",
        "URI": "http://acs.amazonaws.com/groups/global/AuthenticatedUsers"
      },
      "Permission": "WRITE"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.authenticated_write`. The extractor checks whether any ACL grant gives `WRITE` or `FULL_CONTROL` to the `AuthenticatedUsers` group.

```bash
mkdir -p observations

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "ingest-pipeline",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "ingest-pipeline",
        access: {
          authenticated_write: (
            [.Grants[] |
              select(
                (.Permission == "WRITE" or .Permission == "FULL_CONTROL")
                and .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AuthenticatedUsers"))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat acl.json | jq '{
```

**AWS doc basis for the fix**

Remove the `AuthenticatedUsers` `WRITE` grant from the bucket ACL. Use specific account principals in bucket policy if write access is needed. AWS recommends disabling ACLs entirely with **Bucket owner enforced** object ownership.
