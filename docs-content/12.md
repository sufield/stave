## No Wildcard Action Policies

**Control:** CTL.S3.ACCESS.002 (high)

**Misconfiguration:** The bucket policy uses a wildcard action such as `Action: "s3:*"` or `Action: "*"`, which grants far broader permissions than needed and can include read, write, delete, and policy-changing operations.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket analytics
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"WildcardS3Access\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppRole\"},\"Action\":\"s3:*\",\"Resource\":[\"arn:aws:s3:::analytics\",\"arn:aws:s3:::analytics/*\"]}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.has_wildcard_policy`. The extractor parses the bucket policy and checks whether any statement uses `s3:*` or `*` as the action.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "analytics",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "analytics",
        access: {
          has_wildcard_policy: (
            [$p.Statement[] |
              select(
                .Action == "s3:*" or .Action == "*"
                or ((.Action | type) == "array"
                  and (.Action | any(. == "s3:*" or . == "*")))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat policy.json | jq '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
```

**Detect with Stave**

**Fixed AWS CLI output**

The wildcard action is replaced with specific S3 actions. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"ScopedReadOnlyAccess\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppRole\"},\"Action\":[\"s3:GetObject\",\"s3:ListBucket\"],\"Resource\":[\"arn:aws:s3:::analytics\",\"arn:aws:s3:::analytics/*\"]}]}"
}
EOF
```

**AWS doc basis for the fix**

Replace `Action: "s3:*"` or `Action: "*"` with only the exact actions the principal needs. AWS's S3 documentation recommends expressing permissions through explicit actions.

```bash
aws s3api get-bucket-policy --bucket analytics --query Policy --output text > policy.json
# Edit policy.json to replace wildcard with specific actions
aws s3api put-bucket-policy --bucket analytics --policy file://policy.json
```
