## No Authenticated-Users Read Access

**Control:** CTL.S3.AUTH.READ.001 (high)

**Misconfiguration:** The bucket ACL grants `READ` to the `AuthenticatedUsers` group, which means any AWS account holder worldwide can read objects. AWS documents that `AuthenticatedUsers` means any authenticated AWS user, not "only our internal accounts."

**Raw AWS CLI command**

```bash
aws s3api get-bucket-acl --bucket shared-reports
```

**Bad AWS CLI output**

Save as `acl.json`:

```bash
cat > acl.json <<'EOF'
{
  "Owner": {
    "DisplayName": "bucket-owner",
    "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
  },
  "Grants": [
    {
      "Grantee": {
        "Type": "CanonicalUser",
        "DisplayName": "bucket-owner",
        "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
      },
      "Permission": "FULL_CONTROL"
    },
    {
      "Grantee": {
        "Type": "Group",
        "URI": "http://acs.amazonaws.com/groups/global/AuthenticatedUsers"
      },
      "Permission": "READ"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.authenticated_read`. The extractor checks whether any ACL grant gives `READ` or `FULL_CONTROL` to the `AuthenticatedUsers` group.

```bash
mkdir -p observations

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "shared-reports",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "shared-reports",
        access: {
          authenticated_read: (
            [.Grants[] |
              select(
                (.Permission == "READ" or .Permission == "FULL_CONTROL")
                and .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AuthenticatedUsers"))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat acl.json | jq '{
  schema_version: "obs.v0.1",
```

**AWS doc basis for the fix**

Remove the `AuthenticatedUsers` `READ` grant from the bucket ACL. Replace with specific IAM principals in bucket policy if access is needed. AWS recommends keeping ACLs disabled for most modern S3 use cases.
