## Data Classification Tag Required

**Control:** CTL.S3.GOVERNANCE.001 (low)

**Misconfiguration:** The bucket is missing the `data-classification` tag, so controls that depend on sensitivity labels such as `phi`, `pii`, `confidential`, `internal`, or `public` cannot evaluate the bucket correctly. Amazon S3 bucket tags are key-value metadata, and `GetBucketTagging` returns a special `NoSuchTagSet` error when the bucket has no tag set at all.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-tagging --bucket customer-data
```

**Bad AWS CLI output**

When the bucket has no tags, the CLI returns an error. Save an empty tag set as `tagging.json`:

```bash
cat > tagging.json <<'EOF'
{
  "TagSet": []
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind` and whether `properties.storage.tags.data-classification` exists. The extractor maps AWS tags from the `Key`/`Value` array into a flat object.

```bash
mkdir -p observations

cat tagging.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "customer-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "customer-data",
        tags: (
          [.TagSet[] | {(.Key): .Value}] | add // {}
        )
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat tagging.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "customer-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "customer-data",
        tags: (
          [.TagSet[] | {(.Key): .Value}] | add // {}
        )
      }
    }
  }]
}' > observations/2026-03-21T000000Z.json
```

When the `TagSet` is empty, `tags` becomes `{}` — the `data-classification` key is missing, which triggers the control.

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Save as `tagging_fixed.json`:

```bash
cat > tagging_fixed.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "confidential"
    }
  ]
}
EOF
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-tagging \
  --bucket customer-data \
  --tagging '{
    "TagSet": [
      {
        "Key": "data-classification",
        "Value": "confidential"
      }
    ]
  }'
```
