## Shared-Bucket Tenant Isolation Must Enforce Prefix

**Control:** CTL.S3.TENANT.ISOLATION.001 (high)

**Misconfiguration:** A multi-tenant bucket uses prefix-based isolation, but an app-signer identity that produces presigned URLs allows path traversal or disables prefix enforcement. This lets one tenant read or overwrite another tenant's objects.

**Source data**

This control evaluates bucket tags alongside application-level signer identities. The bucket is tagged as shared-tenant, and each signer's configuration is audited. Save as `bucket.json` and `signers.json`:

```bash
cat > bucket.json <<'EOF'
{
  "TagSet": [
    { "Key": "tenant_mode", "Value": "shared" },
    { "Key": "tenant_prefix", "Value": "org-123" }
  ]
}
EOF

cat > signers.json <<'EOF'
[
  {
    "signer_id": "appsigner:s3:uploads",
    "enforce_prefix": false,
    "allow_traversal": true
  }
]
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind eq "bucket"`, `properties.storage.tags.tenant_mode eq "shared"`, `properties.storage.tags.tenant_prefix` is present, and uses `identities` `any_match` to find app signers with `allow_traversal=true` or `enforce_prefix=false` in their purpose.

Identities go at the observation root level (not inside assets):

```bash
mkdir -p observations

jq -n \
  --argjson tags "$(cat bucket.json)" \
  --argjson signers "$(cat signers.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "tenant-data",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "tenant-data",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {})
      }
    }
  }],
  identities: [
    $signers[] | {
      id: .signer_id,
      type: "app_signer",
      vendor: "aws",
      properties: {
        purpose: "enforce_prefix=\(.enforce_prefix) allow_traversal=\(.allow_traversal)"
      }
    }
  ]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**AWS doc basis for the fix**

Enforce tenant isolation with prefix-scoped object ARNs and `${aws:PrincipalTag/tenant-id}`:

```json
{
  "Statement": [
    {
      "Sid": "TenantAccessOwnObjectsOnly",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::123456789012:role/AppTenantRole"},
      "Action": ["s3:GetObject", "s3:PutObject"],
      "Resource": "arn:aws:s3:::tenant-data/tenants/${aws:PrincipalTag/tenant-id}/*"
    }
  ]
}
```

Additionally, ensure all app signers enforce the tenant prefix and block path traversal (`../`) in presigned URLs.
