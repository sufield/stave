## Protected Prefixes Must Not Be Publicly Readable

**Control:** CTL.S3.PUBLIC.PREFIX.001 (high)

**Misconfiguration:** Specific object prefixes such as `invoices/` or `secrets/` are publicly readable via bucket policy. Even when a bucket intentionally hosts some public content under `images/` or `static/`, protected prefixes should never be exposed.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket mixed-content
```

**Bad AWS CLI output**

The policy grants public read on protected prefixes. Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadInvoices\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":[\"arn:aws:s3:::mixed-content/invoices/*\",\"arn:aws:s3:::mixed-content/secrets/*\"]}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control uses `type: prefix_exposure` and evaluates per-prefix exposure from `properties.storage.prefix_exposure`. The extractor parses the policy to determine which prefixes are publicly readable, and provides evidence fields the evaluator needs.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
  ([$p.Statement[] |
    select(.Effect == "Allow" and (.Principal == "*" or .Principal.AWS == "*")) |
    (if (.Resource | type) == "array" then .Resource[] else .Resource end) |
    capture("arn:aws:s3:::[^/]+/(?<prefix>.+)/\\*$") | .prefix + "/"
  ]) as $scopes |
  ($scopes | map({(.): ([$p.Statement[] | select(.Effect == "Allow" and (.Principal == "*" or .Principal.AWS == "*")) | .Sid // "unknown"][0])}) | add // {}) as $source_map |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "mixed-content",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "mixed-content",
        prefix_exposure: {
          has_identity_evidence: true,
          has_resource_evidence: true,
          identity_read_scopes: $scopes,
          identity_source_by_scope: $source_map,
          identity_read_blocked: false,
          resource_read_all: false,
          resource_read_blocked: false
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

The control's `params.protected_prefixes` includes `invoices/` and `secrets/`, so both trigger violations.

**Fixed AWS CLI output**

Public read is limited to an allowed prefix (`images/`). Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadImages\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::mixed-content/images/*\"},{\"Sid\":\"InternalRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppRole\"},\"Action\":\"s3:GetObject\",\"Resource\":[\"arn:aws:s3:::mixed-content/invoices/*\",\"arn:aws:s3:::mixed-content/secrets/*\"]}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove public `s3:GetObject` grants for protected prefixes. Keep public access only on intentionally public prefixes that match `params.allowed_public_prefixes` in the control.
