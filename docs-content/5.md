## Versioning Required

**Control:** CTL.S3.VERSION.001 (medium)

**Misconfiguration:** Versioning is not enabled for the bucket, so overwritten or deleted objects cannot be recovered through S3 object versions. AWS says versioning keeps multiple variants of an object in the same bucket and helps you recover from unintended user actions and application failures.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-versioning --bucket backups
```

**Bad AWS CLI output**

AWS documents that if versioning has never been set on a bucket, `GetBucketVersioning` does not return a versioning state value. Save as `versioning.json`:

```bash
cat > versioning.json <<'EOF'
{}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.kind` and `properties.storage.versioning.enabled`. The extractor derives `enabled` from whether `Status` equals `"Enabled"`.

```bash
mkdir -p observations

cat versioning.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "backups",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "backups",
        versioning: {
          enabled: (.Status == "Enabled")
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat versioning.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-21T00:00:00Z",
  assets: [{
    id: "backups",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "backups",
        versioning: {
          enabled: (.Status == "Enabled")
        }
      }
    }
  }]
}' > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

Save as `versioning_fixed.json`:

```bash
cat > versioning_fixed.json <<'EOF'
{
  "Status": "Enabled"
}
EOF
```

**Confirm remediation**

```bash
mkdir -p observations-after

cat versioning_fixed.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-22T00:00:00Z",
  assets: [{
    id: "backups",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "backups",
        versioning: {
          enabled: (.Status == "Enabled")
        }
      }
    }
  }]
}' > observations-after/2026-03-22T000000Z.json

stave apply \
  --observations observations-after \
  --max-unsafe 12h \
  --now 2026-03-22T12:00:00Z
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-versioning \
  --bucket backups \
  --versioning-configuration Status=Enabled
```
