## No Public Read via Policy

**Control:** CTL.S3.PUBLIC.007 (critical)

**Misconfiguration:** The bucket policy explicitly grants `s3:GetObject` to `Principal: "*"`, which makes objects publicly readable. AWS's S3 policy examples use this exact pattern for public read access.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-policy --bucket shared-assets
```

**Bad AWS CLI output**

Save as `policy.json`:

```bash
cat > policy.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"PublicReadGetObject\",\"Effect\":\"Allow\",\"Principal\":\"*\",\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::shared-assets/*\"}]}"
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.read_via_identity`. The extractor parses the bucket policy and checks whether any `Allow` statement grants read access to `Principal: "*"`.

```bash
mkdir -p observations

cat policy.json | jq '
  (.Policy | fromjson) as $p |
{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "shared-assets",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "shared-assets",
        access: {
          read_via_identity: (
            [$p.Statement[] |
              select(
                .Effect == "Allow"
                and (.Principal == "*" or .Principal.AWS == "*")
                and (.Action == "s3:GetObject" or .Action == "s3:*"
                  or ((.Action | type) == "array"
                    and (.Action | any(. == "s3:GetObject" or . == "s3:*"))))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json
```

**Detect with Stave**

**Fixed AWS CLI output**

The public-read statement is replaced with access limited to a specific IAM role. Save as `policy_fixed.json`:

```bash
cat > policy_fixed.json <<'EOF'
{
  "Policy": "{\"Version\":\"2012-10-17\",\"Statement\":[{\"Sid\":\"AllowAppRead\",\"Effect\":\"Allow\",\"Principal\":{\"AWS\":\"arn:aws:iam::123456789012:role/AppReadRole\"},\"Action\":\"s3:GetObject\",\"Resource\":\"arn:aws:s3:::shared-assets/*\"}]}"
}
EOF
```

**AWS doc basis for the fix**

Remove the statement that grants `s3:GetObject` to `Principal: "*"`, or replace it with a statement that grants access only to a specific AWS principal. Use `get-bucket-policy-status` to verify the bucket is no longer considered public.
