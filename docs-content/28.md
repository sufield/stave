## PHI Buckets Must Use SSE-KMS with Customer-Managed Key

**Control:** CTL.S3.ENCRYPT.003 (high)

**Misconfiguration:** The bucket is tagged `data-classification: phi` but uses SSE-S3 (`AES256`) instead of SSE-KMS with a customer-managed key. PHI data requires explicit KMS key control for key rotation, access policies, and audit logging.

**Raw AWS CLI commands**

```bash
aws s3api get-bucket-encryption --bucket phi-records
aws s3api get-bucket-tagging --bucket phi-records
```

**Bad AWS CLI output**

SSE-S3 encryption on a PHI bucket. Save as `encryption.json` and `tagging.json`:

```bash
cat > encryption.json <<'EOF'
{
  "ServerSideEncryptionConfiguration": {
    "Rules": [
      {
        "ApplyServerSideEncryptionByDefault": {
          "SSEAlgorithm": "AES256"
        }
      }
    ]
  }
}
EOF

cat > tagging.json <<'EOF'
{
  "TagSet": [
    {
      "Key": "data-classification",
      "Value": "phi"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.tags.data-classification eq "phi"` AND (`properties.storage.encryption.algorithm ne "aws:kms"` OR `properties.storage.encryption.kms_key_id eq ""`).

```bash
mkdir -p observations

jq -n \
  --argjson enc "$(cat encryption.json)" \
  --argjson tags "$(cat tagging.json)" \
'{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "phi-records",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "phi-records",
        tags: ([$tags.TagSet[] | {(.Key): .Value}] | add // {}),
        encryption: {
          algorithm: ($enc.ServerSideEncryptionConfiguration.Rules[0].ApplyServerSideEncryptionByDefault.SSEAlgorithm // ""),
          kms_key_id: ($enc.ServerSideEncryptionConfiguration.Rules[0].ApplyServerSideEncryptionByDefault.KMSMasterKeyID // "")
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

jq '.captured_at = "2026-03-21T00:00:00Z"' observations/2026-03-20T000000Z.json > observations/2026-03-21T000000Z.json
```

**Detect with Stave**

```bash
stave init --profile aws-s3

stave apply \
  --observations observations \
  --max-unsafe 12h \
  --now 2026-03-21T12:00:00Z
```

**Fixed AWS CLI output**

SSE-KMS with a customer-managed key. Save as `encryption_fixed.json`:

```bash
cat > encryption_fixed.json <<'EOF'
{
  "ServerSideEncryptionConfiguration": {
    "Rules": [
      {
        "ApplyServerSideEncryptionByDefault": {
          "SSEAlgorithm": "aws:kms",
          "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789012:key/11111111-2222-3333-4444-555555555555"
        },
        "BucketKeyEnabled": true
      }
    ]
  }
}
EOF
```

**AWS doc basis for the fix**

```bash
aws s3api put-bucket-encryption \
  --bucket phi-records \
  --server-side-encryption-configuration '{
    "Rules": [
      {
        "ApplyServerSideEncryptionByDefault": {
          "SSEAlgorithm": "aws:kms",
          "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789012:key/11111111-2222-3333-4444-555555555555"
        },
        "BucketKeyEnabled": true
      }
    ]
  }'
```
