## No FULL_CONTROL ACL Grants to Public

**Control:** CTL.S3.ACL.FULLCONTROL.001 (critical)

**Misconfiguration:** The bucket ACL grants `FULL_CONTROL` to the `AllUsers` group, which gives anyone on the internet complete access including read, write, delete, and ACL modification. This is the worst-case ACL misconfiguration.

**Raw AWS CLI command**

```bash
aws s3api get-bucket-acl --bucket open-bucket
```

**Bad AWS CLI output**

Save as `acl.json`:

```bash
cat > acl.json <<'EOF'
{
  "Owner": {
    "DisplayName": "bucket-owner",
    "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
  },
  "Grants": [
    {
      "Grantee": {
        "Type": "CanonicalUser",
        "DisplayName": "bucket-owner",
        "ID": "1111222233334444555566667777888899990000aaaabbbbccccddddeeeeffff"
      },
      "Permission": "FULL_CONTROL"
    },
    {
      "Grantee": {
        "Type": "Group",
        "URI": "http://acs.amazonaws.com/groups/global/AllUsers"
      },
      "Permission": "FULL_CONTROL"
    }
  ]
}
EOF
```

**Extract to a Stave observation (`obs.v0.1`)**

The control checks `properties.storage.access.has_full_control_public`. The extractor checks whether any ACL grant gives `FULL_CONTROL` to the `AllUsers` group.

```bash
mkdir -p observations

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
  captured_at: "2026-03-20T00:00:00Z",
  assets: [{
    id: "open-bucket",
    type: "aws_s3_bucket",
    vendor: "aws",
    properties: {
      storage: {
        kind: "bucket",
        name: "open-bucket",
        access: {
          has_full_control_public: (
            [.Grants[] |
              select(
                .Permission == "FULL_CONTROL"
                and .Grantee.Type == "Group"
                and (.Grantee.URI | contains("AllUsers"))
              )
            ] | length > 0
          )
        }
      }
    }
  }]
}' > observations/2026-03-20T000000Z.json

cat acl.json | jq '{
  schema_version: "obs.v0.1",
  generated_by: { source_type: "aws-s3-snapshot", tool: "aws-cli", tool_version: "2.x" },
```

**Detect with Stave**

**AWS doc basis for the fix**

Remove the `AllUsers` `FULL_CONTROL` grant from the bucket ACL, or reset to private. AWS recommends using **Bucket owner enforced** object ownership to disable ACLs entirely.
