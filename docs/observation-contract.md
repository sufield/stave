# Observation Contract

## What an observation is

An observation is a point-in-time, normalized snapshot of resource state used by Stave evaluation.

Current schema: `obs.v0.1`

Observations are generated by extractors (for example `stave ingest --profile mvp1-s3`) or trusted pipelines.
Users should not author observations manually except for tests/debug reproductions.

## Versioning policy

- Contract version is encoded in `schema_version`.
- Breaking changes require a new schema version.
- Non-breaking additions may occur only when they do not invalidate existing consumers.

For MVP 1.0, `obs.v0.1` is treated as stable for supported S3 fields.

## MVP 1.0 stability promise

For S3 public exposure evaluation, these keys are the stability surface:

- `schema_version`
- `captured_at`
- `resources[].id`, `resources[].type`, `resources[].vendor`
- `resources[].properties.storage.visibility.*`
- `resources[].properties.storage.controls.public_access_block.*`
- `resources[].properties.storage.encryption.*`
- `resources[].properties.storage.versioning.*`
- `resources[].properties.storage.logging.*`
- `resources[].properties.storage.access.*`
- `resources[].properties.storage.object_lock.*`

## Field dictionary (customer-relevant)

| Field | Meaning | derived_from | Type | Example |
|---|---|---|---|---|
| `schema_version` | Observation schema contract version | extractor output envelope | string | `obs.v0.1` |
| `captured_at` | Snapshot capture time in RFC3339 | extractor clock / source timestamp | string (date-time) | `2026-01-11T00:00:00Z` |
| `resources[].id` | Stable resource identifier for findings | source inventory identity | string | `arn:aws:s3:::billing-phi` |
| `resources[].properties.storage.visibility.public_read` | Effective public read exposure | policy + ACL + PAB normalization | bool | `true` |
| `resources[].properties.storage.visibility.public_list` | Effective public list exposure | policy + ACL + PAB normalization | bool | `false` |
| `resources[].properties.storage.visibility.public_write` | Effective public write exposure | ACL/policy normalization | bool | `false` |
| `resources[].properties.storage.visibility.public_read_via_policy` | Public read specifically via policy path | bucket policy analysis | bool | `true` |
| `resources[].properties.storage.visibility.public_read_via_acl` | Public read specifically via ACL path | ACL grant analysis | bool | `false` |
| `resources[].properties.storage.controls.public_access_fully_blocked` | Whether effective PAB blocks all public access controls | PAB aggregation | bool | `false` |
| `resources[].properties.storage.controls.public_access_block.block_public_acls` | S3 Block Public ACLs setting | get-public-access-block | bool | `true` |
| `resources[].properties.storage.encryption.at_rest_enabled` | Whether at-rest encryption is enabled | bucket encryption config | bool | `true` |
| `resources[].properties.storage.encryption.algorithm` | Effective encryption algorithm | SSE config normalization | string | `aws:kms` |
| `resources[].properties.storage.versioning.enabled` | Versioning enabled status | get-bucket-versioning | bool | `true` |
| `resources[].properties.storage.versioning.mfa_delete_enabled` | MFA delete status (if available) | versioning controls | bool | `false` |
| `resources[].properties.storage.logging.enabled` | Access logging enabled status | logging configuration | bool | `true` |
| `resources[].properties.storage.access.has_external_access` | Whether external accounts can access | policy principal analysis | bool | `false` |
| `resources[].properties.storage.object_lock.enabled` | Object lock enablement status | object lock configuration | bool | `false` |

## Gaps

- Some fields exist for specific controls but are not yet centralized in one canonical "field catalog" page.
- Source-specific nuances (for example partial evidence availability) are spread across command docs and extractor logic.

## Important warning

Do not hand-author production observations.
Generate observations via extractor commands and treat manual JSON edits as test-only.
