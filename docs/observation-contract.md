# Observation Contract

An observation is a point-in-time, normalized snapshot of asset state used by
Stave evaluation. Current schema: `obs.v0.1`.

Observations are generated by external extractors (any language producing
`obs.v0.1` JSON) or trusted pipelines. Do not hand-author production
observations — treat manual JSON edits as test-only. See
[Building an Extractor](extractor-prompt.md) for a jumpstart template.

## Versioning policy

- Contract version is encoded in `schema_version`.
- Breaking changes require a new schema version.
- Non-breaking additions may occur only when they do not invalidate
  existing consumers.

For MVP 1.0, `obs.v0.1` is treated as stable for supported S3 fields.

---

## Envelope

Every observation file is a flat JSON object (no `"snapshots"` wrapper —
the schema rejects `additionalProperties`).

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `schema_version` | string | yes | `"obs.v0.1"` |
| `captured_at` | string (RFC 3339) | yes | Snapshot capture time |
| `assets` | array | yes | Asset state objects (see below) |
| `generated_by` | object | no | Extraction metadata |
| `identities` | array | no | IAM identity objects (see below) |

### `generated_by`

| Field | Type | Description |
|-------|------|-------------|
| `source_type` | string | Extractor kind (`"aws-s3-snapshot"`, `"terraform.plan_json"`) |
| `tool` | string | Tool name (`"stave-extract"`) |
| `tool_version` | string | Semantic version |
| `provider` | string | Cloud provider (`"aws"`) |
| `provider_version` | string | Provider API version |

---

## Asset structure

Each entry in `assets[]`:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Stable identifier (ARN or synthetic ID) |
| `type` | string | yes | Asset type (`"storage_bucket"`, pattern `^[a-z0-9][a-z0-9_.-]*$`) |
| `vendor` | string | yes | Cloud vendor (`"aws"`) |
| `properties` | object | yes | Domain-specific state (see field dictionary) |
| `source` | object | no | Source reference (`file`, `line`) |

---

## Identity structure

Each entry in `identities[]` (optional, used by ACL/policy controls):

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | Principal identifier (ARN, email) |
| `type` | string | yes | Identity type (pattern `^[a-z0-9][a-z0-9_.-]*$`) |
| `vendor` | string | yes | Cloud vendor |
| `properties` | object | yes | Identity-specific properties |

---

## MVP 1.0 stability promise

For S3 evaluation, these property namespaces are the stability surface.
Fields within these namespaces will not be removed or have their semantics
changed without a schema version bump.

- `storage.visibility.*`
- `storage.controls.*`
- `storage.encryption.*`
- `storage.versioning.*`
- `storage.logging.*`
- `storage.access.*`
- `storage.object_lock.*`
- `storage.website.*`
- `storage.lifecycle.*`
- `storage.content.*`
- `storage.tags.*`
- `cdn.*`
- `s3_ref.*`
- `s3_upload.*`

## Field dictionary
