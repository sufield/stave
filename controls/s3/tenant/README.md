# S3 Tenant Isolation Controls

Controls in this directory detect prefix-based tenant isolation failures in shared S3 buckets.

| ID | Name | What it checks |
|----|------|----------------|
| CTL.S3.TENANT.ISOLATION.001 | Shared-Bucket Tenant Isolation Must Enforce Prefix | App-signer identity allows path traversal or disables prefix enforcement |

## Attack Pattern

Multi-tenant applications often use a single S3 bucket with prefix-based isolation (e.g., `org-123/uploads/`). Presigned URLs generated by app-signer identities must bind to the tenant's prefix. Two failure modes:

1. **Path traversal enabled (`allow_traversal=true`):** The signer allows `../` in object keys, letting one tenant escape their prefix and access another tenant's objects.

2. **Prefix enforcement disabled (`enforce_prefix=false`):** The signer does not restrict the key to the tenant prefix, allowing writes to arbitrary paths in the bucket.

## Predicate Logic

TENANT.ISOLATION.001 uses a compound predicate with nested `any_match` on identities:

1. Bucket must be `kind == "bucket"`
2. Tagged `tenant_mode == "shared"` with `tenant_prefix` present
3. Any identity of type `app_signer` with an `appsigner:s3:` ID must have either `allow_traversal=true` or `enforce_prefix=false` in its purpose field
